A collapsible accordion component that supports dynamic state and different variants.

## Usage

<Tabs>
  <TabItem label="Demo">
  <AccordionDemo id="test1" variant="outlined" client:load />
</TabItem>
  <TabItem label="Code">
    ```svelte
      <script lang="ts">
        import { Accordion } from "@dxdns/feflow"
      </script>

      <Accordion
        id="test1"
        variant="text"
        label="test1"
      >
        <span>test</span>
      </Accordion>
      ```
  </TabItem>
</Tabs>

<br />
<br />

| Prop      | Type                   | Default       | Description                                                   |
|-----------|------------------------|---------------|---------------------------------------------------------------|
| `variant` | `"text"` \| `"outlined"`| `"text"`      | Sets the visual style of the accordion (e.g., text or outlined).|
| `label`   | `string`                | `null`        | Label for the accordion header.                               |
| `isOpen`  | `boolean`               | `false`       | Controls whether the accordion is open or closed.             |
| `onclick` | `function`              | `null`        | Callback function when the accordion is clicked.              |

| Slot     | Description                                      |
|----------|--------------------------------------------------|
| default  | Content inside the accordion, typically a section of collapsible content. |

# Accessibility
This component uses native `<button>` and `<div>` elements to manage the interactive accordion state and is compatible with standard accessibility features like `aria-expanded`.

# Best Practices
- Use `variant="outlined"` for primary or emphasized accordions.
- Ensure the label is descriptive for screen readers.
- Keep the accordion sections concise and easy to navigate with keyboard events.
